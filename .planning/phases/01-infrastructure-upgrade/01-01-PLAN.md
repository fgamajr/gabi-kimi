  <action>
**Step A: Update ops/setup_elasticsearch.sh**

Update the docker run command to use 4GB heap instead of 512MB. Change the `ES_JAVA_OPTS` line from `-Xms512m -Xmx512m` to `-Xms4g -Xmx4g`. Keep everything else identical (same image 8.15.4, same volume /media/psf/gabi_es, same config flags).

**Step B: Create src/backend/search/es_index_v2.json**

Copy the entire v1 mapping from `es_index_v1.json` and add three new fields to `mappings.properties`:

```json
"embedding": {
  "type": "dense_vector",
  "dims": 1024,
  "index": true,
  "similarity": "cosine",
  "index_options": { "type": "int8_hnsw" }
},
"embedding_status": { "type": "keyword" },
"embedding_model": { "type": "keyword" }
```

Also add `"refresh_interval": "1s"` to `settings.index` (explicit, matching v1 default). Keep `number_of_shards: 1`, `number_of_replicas: 0`, same `pt_folded` analyzer.

IMPORTANT: `"index": true` MUST be explicit on the dense_vector field — omitting it creates a stored-only field that rejects kNN queries.

**Step C: Recreate Docker container with new heap**

1. Pre-flight check: read /proc/meminfo to confirm VM has >= 8GB RAM available
2. Stop and remove the existing gabi-es container: `docker stop gabi-es && docker rm gabi-es`
3. Recreate with the updated command from setup_elasticsearch.sh:
   ```
   docker run -d --name gabi-es -p 9200:9200 \
     -e discovery.type=single-node \
     -e xpack.security.enabled=false \
     -e "ES_JAVA_OPTS=-Xms4g -Xmx4g" \
     -v /media/psf/gabi_es:/usr/share/elasticsearch/data \
     docker.elastic.co/elasticsearch/elasticsearch:8.15.4
   ```
4. Wait for ES to be ready (poll localhost:9200 up to 60 seconds — ES with 4GB heap takes longer to start)
5. Verify heap: `GET /_nodes/stats/jvm` — heap_max_in_bytes >= 4294967296

**Step D: Create v2 index in ES**

1. Load es_index_v2.json from disk
2. PUT http://localhost:9200/gabi_documents_v2 with the mapping body
3. Verify: `GET /gabi_documents_v2/_mapping` confirms embedding field exists with type dense_vector

Do NOT write any documents to v2 yet — that happens in Plan 02 via _reindex.
