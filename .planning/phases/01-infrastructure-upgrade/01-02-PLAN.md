  <action>
This task operates entirely via ES REST APIs — no files are created or modified.

**Step A: Pre-flight checks**

1. Confirm ES is healthy: `GET /_cluster/health` — status must be green or yellow
2. Confirm v2 index exists and is empty: `GET /gabi_documents_v2/_count` — count should be 0
3. Get v1 document count for later comparison: `GET /gabi_documents_v1/_count` — store this value

**Step B: Optimize v2 for bulk loading**

Disable refresh during reindex for performance:
```
PUT /gabi_documents_v2/_settings
{"index.refresh_interval": "-1"}
```

**Step C: Launch async reindex**

```
POST /_reindex?wait_for_completion=false&slices=auto
{
  "source": {"index": "gabi_documents_v1"},
  "dest": {"index": "gabi_documents_v2"}
}
```

Store the returned task ID.

IMPORTANT: Use `wait_for_completion=false` — the reindex of ~7M docs will take 2-4 hours and will timeout the HTTP connection if synchronous.

**Step D: Monitor reindex progress**

Poll `GET /_tasks/{task_id}` every 30 seconds. Log progress (created count vs total). Wait until `completed: true`.

If the task reports failures, log them but continue — partial failures on individual documents should be investigated but don't block the overall reindex.

**Step E: Restore v2 settings**

After reindex completes:
```
PUT /gabi_documents_v2/_settings
{"index.refresh_interval": "1s"}
```

**Step F: Verify document count parity**

```python
v1_count = GET /gabi_documents_v1/_count -> count
v2_count = GET /gabi_documents_v2/_count -> count
assert v1_count == v2_count
```

If counts don't match, log the difference and report it — do NOT proceed to alias swap.

**Step G: Verify BM25 search works on v2**

Run a sample BM25 query directly against v2:
```
GET /gabi_documents_v2/_search?q=decreto&size=1
```
Confirm hits.total.value > 0. This proves BM25 functionality is intact on the new index.
