**snapshots.py** — ES snapshot management for Tigris backup (per user decision: "Daily ES snapshots to Tigris (S3-compatible)"):

Using the code example from 11-RESEARCH.md:

1. `async def register_snapshot_repo(es_url: str)`:
   - PUT `{es_url}/_snapshot/tigris_backup` with body:
     ```json
     {
       "type": "s3",
       "settings": {
         "bucket": "gabi-dou-es-snapshots",
         "endpoint": "fly.storage.tigris.dev",
         "protocol": "https",
         "path_style_access": true
       }
     }
     ```
   - Called once during worker startup (lifespan)
   - Log success/failure; do not crash worker if Tigris is unreachable (warn and continue)

2. `async def create_snapshot(es_url: str)`:
   - Create a named snapshot: PUT `{es_url}/_snapshot/tigris_backup/daily-{YYYYMMDD-HHMM}`
   - Body: `{"indices": "gabi_documents_v1", "include_global_state": false}`
   - Log result; if snapshot fails, log error but do not crash

**scheduler.py** — APScheduler cron job orchestration:

1. Import AsyncIOScheduler from apscheduler.schedulers.asyncio, CronTrigger from apscheduler.triggers.cron
2. Import all pipeline modules: run_discovery, run_download, run_extract, run_ingest, run_verify
3. Import create_snapshot from snapshots
4. Import Registry

5. Create module-level `scheduler = AsyncIOScheduler(timezone="UTC")`
6. Create module-level `_paused = False` flag
7. Create `PHASE_MAP` dict mapping phase names to functions:
   ```python
   PHASE_MAP = {
       "discovery": run_discovery,
       "download": run_download,
       "extract": run_extract,
       "ingest": run_ingest,
       "verify": run_verify,
   }
   ```

8. Create wrapper functions for each cron job that:
   - Check `_paused` flag (skip if paused, log skip)
   - Create a Registry instance
   - Create a pipeline_run record via `registry.create_pipeline_run(phase)`
   - Call the corresponding pipeline function with registry, run_id, and es_url (from env)
   - Complete the pipeline_run with stats
   - Log errors and mark run as failed on exception

9. `def configure_scheduler()` — adds 6 cron jobs + snapshot:
   - discovery: hour=23, minute=0
   - download: hour=23, minute=30
   - extract: hour=23, minute=45 (added — needed between download and ingest)
   - ingest: hour=0, minute=0
   - verify: hour=1, minute=0
   - retry: hour=6, minute=0 (calls a retry function that finds failed files with retry_count < 3, resets to QUEUED)
   - snapshot: hour=2, minute=0 (daily ES snapshot to Tigris via create_snapshot)
   - Each job has `id=phase_name`, `replace_existing=True`, `max_instances=1`

10. `def pause_scheduler()` / `def resume_scheduler()` — toggle `_paused` flag
11. `def get_scheduler_status()` -> dict with running, paused, jobs (list of {id, next_run_time})
12. `async def trigger_phase(phase: str)` — validate phase name, run the wrapper function immediately via scheduler.add_job() with no trigger (runs once now)

**main.py** — Worker entrypoint:

1. Create FastAPI app with title "GABI Worker Internal API"
2. Lifespan context manager:
   - On startup: init Registry (create schema), register_snapshot_repo(es_url), configure_scheduler(), scheduler.start(), log "Worker started"
   - Record start_time for uptime calculation
   - On shutdown: scheduler.shutdown(wait=False), log "Worker stopped"
3. Include API router from api.py
4. Add heartbeat: scheduler job every 60s that logs "heartbeat" (for dashboard health monitoring)
5. Bottom: `if __name__ == "__main__": uvicorn.run("src.backend.worker.main:app", host="0.0.0.0", port=8081)`

Create `src/backend/worker/api.py` with FastAPI APIRouter:

**Registry endpoints:**
- `GET /registry/status` — returns dict of status -> count from registry.get_status_counts()
- `GET /registry/months?year={year}` — returns month/file data from registry.get_months(year)
- `GET /registry/files/{file_id}` — returns single file record from registry.get_file(file_id), 404 if not found

**Pipeline control endpoints:**
- `GET /pipeline/runs?limit=50` — returns recent pipeline runs from registry.get_pipeline_runs(limit)
- `GET /pipeline/logs?run_id=&file_id=&level=&limit=200` — returns filtered logs from registry.get_logs(...)
- `POST /pipeline/trigger/{phase}` — validates phase name against PHASE_MAP keys, calls trigger_phase(), returns {"triggered": phase}
- `POST /pipeline/retry/{file_id}` — calls registry.retry_file(file_id), returns {"retried": file_id}, 400 if max retries exceeded
- `POST /pipeline/pause` — calls pause_scheduler(), returns {"paused": true}
- `POST /pipeline/resume` — calls resume_scheduler(), returns {"paused": false}

**Health endpoint:**
- `GET /health` — returns {"status": "ok", "uptime_seconds": N, "scheduler_running": bool, "scheduler_paused": bool, "last_heartbeat": ISO timestamp, "disk_usage": {...}}

**Implementation notes:**
- Use a module-level `_registry: Registry | None = None` that gets set during lifespan (main.py sets it after init_db)
- Import scheduler status functions from scheduler.py
- All endpoints are async
- Error responses use HTTPException with appropriate status codes
- Registry access uses `async with registry.get_db() as db:` pattern
