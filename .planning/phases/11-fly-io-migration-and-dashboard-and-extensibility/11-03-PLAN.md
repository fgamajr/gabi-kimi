`async def run_discovery(registry: Registry, run_id: str, es_url: str) -> dict`:
1. Reuse Liferay JSONWS API logic from `auto_discovery.py`:
   - Call `/api/jsonws/dlapp/get-folders` on in.gov.br with group_id=49035712 to list month folders
   - For each folder, call `/api/jsonws/dlapp/get-file-entries` to list ZIP files
   - Import constants from `src.backend.ingest.zip_downloader`: `ALL_SECTIONS`, `TAGS_API_URL`, `_build_session`, `_random_ua`, `get_folder_id`
2. For each discovered ZIP file:
   - Parse filename to extract section and year_month (pattern: S{section}{month}{year}.zip)
   - Check if already in registry via `registry.get_file_by_filename(filename)`
   - If new: `registry.insert_file(filename, section, year_month, folder_id, file_url)`
   - Log discovery via `registry.add_log_entry(run_id, file_id, "INFO", "Discovered new file")`
3. Rate limit: use `asyncio.Semaphore(5)` to cap concurrent requests
4. Fallback: If Liferay API is unavailable, use predictive HEAD probes on constructed URLs (reuse `discovery_probe.py` logic)
5. Return stats: `{"new_files": N, "existing_files": M}`

**downloader.py** — Adapt from `src/backend/ingest/zip_downloader.py`:

`async def run_download(registry: Registry, run_id: str) -> dict`:
1. Query registry for files with status QUEUED: `registry.get_files_by_status(FileStatus.QUEUED)`
2. For each file:
   - Update status to DOWNLOADING
   - Download ZIP using requests (sync, with random UA rotation from zip_downloader._random_ua)
   - Save to `/data/tmp/{filename}`
   - Compute SHA256 hash, store in registry
   - Update file_size_bytes
   - Update status to DOWNLOADED
   - On failure: update status to DOWNLOAD_FAILED, set error_message, log error
3. Rate limit: `time.sleep(0.2)` between downloads (max 5 req/s)
4. Return stats: `{"downloaded": N, "failed": M}`

Both functions accept a Registry instance and a run_id for logging. They are designed to be called by the scheduler (Plan 04).

**extractor.py** — New module for ZIP extraction:

`async def run_extract(registry: Registry, run_id: str) -> dict`:
1. Query registry for DOWNLOADED files
2. For each file:
   - Update status to EXTRACTING
   - Open ZIP from `/data/tmp/{filename}`
   - ZIP Slip protection: validate all paths are within `/data/tmp/extract/{filename_stem}/` (reject absolute paths and `..` components)
   - Extract XML files to `/data/tmp/extract/{filename_stem}/`
   - Encoding detection: Use chardet on first 10KB of each XML. If confidence < 0.8, try Latin-1 decode. Log encoding detections.
   - Handle multi-era formats:
     - 2019+ ZIPs: contain XML files directly
     - 2002-2018 ZIPs: may contain nested directories, HTML files, or TXT files alongside XMLs. Extract only .xml files.
   - Update status to EXTRACTED, record doc_count (number of XMLs found)
   - On failure: EXTRACT_FAILED with error_message
3. Return stats: `{"extracted": N, "failed": M, "total_xmls": K}`

**ingestor.py** — Adapt from existing xml_parser + es_indexer:

`async def run_ingest(registry: Registry, run_id: str, es_url: str) -> dict`:
1. Query registry for EXTRACTED files
2. For each file:
   - Update status to INGESTING
   - List XMLs in `/data/tmp/extract/{filename_stem}/`
   - For each XML:
     - Read file with detected encoding (from extractor)
     - Parse using `src.backend.ingest.xml_parser.INLabsXMLParser` (import and reuse)
     - Normalize using `src.backend.ingest.normalizer` (import hash computation)
     - Build ES bulk action: `{"index": {"_index": "gabi_documents_v1", "_id": doc_id}}`
     - doc_id: deterministic from natural_key_hash (sha256 of pub_date + section + name + id_materia)
   - Bulk index to ES via httpx POST to `{es_url}/_bulk` with batch size 300 docs
   - Update status to INGESTED, update doc_count with actual indexed count
   - On failure: INGEST_FAILED with error_message, partial results still indexed
   - Clean up extracted files after successful ingest
3. Return stats: `{"ingested_files": N, "ingested_docs": M, "failed_files": K}`

Document mapping for ES bulk index body:
```python
{
    "pub_date": article.pub_date_iso,  # Convert DD/MM/YYYY to ISO
    "section": article.pub_name,
    "title": article.name,
    "body": article.body_text,
    "act_type": article.art_type,
    "act_number": article.art_number,
    "orgao": article.orgao,
    "signatario": article.signatario,
    "dou_page": article.page,
    "source_zip": filename,
    "natural_key_hash": hash_value,
    "indexed_at": datetime.now(timezone.utc).isoformat(),
}
```

**verifier.py** — Post-ingest verification:

`async def run_verify(registry: Registry, run_id: str, es_url: str) -> dict`:
1. Query registry for INGESTED files (not yet VERIFIED)
2. For each file:

   - If mismatch: update status to VERIFY_FAILED with error_message describing the delta
3. Return stats: `{"verified": N, "failed": M}`
