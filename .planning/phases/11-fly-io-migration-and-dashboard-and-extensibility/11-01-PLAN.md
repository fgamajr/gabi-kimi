  <name>Task 1: Create Elasticsearch machine deployment config</name>
  <files>ops/deploy/es/fly.toml, ops/deploy/es/Dockerfile, ops/deploy/es/entrypoint.sh, ops/deploy/es/elasticsearch.yml</files>
  <action>
Create 4 files for the ES Fly.io machine:

1. `ops/deploy/es/fly.toml`:
   - app = 'gabi-dou-es', primary_region = 'gru'
   - Mount volume `gabi_es_data` at `/usr/share/elasticsearch/data`
   - VM: performance-2x, 4096mb memory
   - No `[http_service]` — ES is internal only, accessed via `.internal` DNS
   - No `[[services]]` — not exposed to internet
   - `auto_stop_machines = false` (ES must stay running)
   - Build dockerfile = 'ops/deploy/es/Dockerfile'

2. `ops/deploy/es/Dockerfile`:
   - FROM docker.elastic.co/elasticsearch/elasticsearch:8.15.2
   - COPY elasticsearch.yml to /usr/share/elasticsearch/config/
   - COPY entrypoint.sh with chmod 755
   - ENV: discovery.type=single-node, xpack.security.enabled=false, ES_JAVA_OPTS="-Xms2g -Xmx2g"
   - USER root (entrypoint handles permission fix then drops to elasticsearch user)
   - ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]
   - EXPOSE 9200

3. `ops/deploy/es/entrypoint.sh`:
   - #!/bin/bash
   - chown -R elasticsearch:elasticsearch /usr/share/elasticsearch/data
   - exec su-exec elasticsearch /usr/share/elasticsearch/bin/elasticsearch
   - Note: ES base image includes su-exec

4. `ops/deploy/es/elasticsearch.yml`:
   - cluster.name: gabi-dou
   - node.name: es-gru-01
   - network.host: 0.0.0.0
   - discovery.type: single-node
   - xpack.security.enabled: false
   - path.data: /usr/share/elasticsearch/data
   - Configure index settings for Portuguese analysis (brazilian_stemmer, asciifolding, brazilian_stop) as index template defaults
  </action>
