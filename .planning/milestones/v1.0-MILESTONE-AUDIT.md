### Security Note

The `/api/worker/*` proxy route lacks backend authentication. The UI gates access via `ProtectedRoute requiredRole="admin"`, but the API endpoint itself is publicly accessible. In production, the worker is only reachable via `.internal` DNS, but the proxy bypasses this. An unauthenticated request to `/api/worker/pipeline/pause` could pause the autonomous pipeline.

**Recommendation:** Add `Depends(require_admin_access)` to the proxy route before production deployment.
