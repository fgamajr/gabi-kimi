# GABI DOU — Blueprint de Frontend v4 (Revisado)

## 1. Conceito Visual: "Brasília Moderna"

Identidade inspirada no racionalismo brasileiro: espaço generoso, tipografia com autoridade, contraste calculado. O dourado do Diário Oficial como acento — não como tema. Nada genérico, nada que pareça template SaaS.

---

## 2. Stack Técnica

| Camada | Escolha | Justificativa |
|---|---|---|
| Framework | React 18 + Vite 5 | Fast refresh, ESM nativo, tree-shaking agressivo |
| Primitives UI | shadcn/ui (Radix) | Copia-se o primitivo, descarta-se o estilo |
| Ícones | Lucide React | Tree-shakeable (~0.5KB/ícone), sem identidade genérica |
| Bottom Sheet | vaul | <5KB gzip, spring physics nativo, acessível |
| Motion | Framer Motion 11 | Apenas para layout animations e page transitions; lazy-loaded em chunk separado |
| Estilos | Tailwind CSS 3.4 + CSS custom properties | Tokens como variáveis, utility classes para layout (v3.4 stable, ver ADR-007) |
| Estado client | Zustand (+ persist middleware com IndexedDB) | Leve, sem boilerplate, async storage via idb-keyval (~600B) |
| Estado server | TanStack Query v5 | Cache, stale-while-revalidate, retry automático, prefetch, AbortSignal |
| Virtualização | react-virtuoso | estimateSize dinâmico para documentos longos, endReached para streaming |
| Busca local | FlexSearch | Busca full-text client-side dentro de documento aberto |
| Strings | `messages/pt-BR.ts` (typed dictionary) | Zero runtime cost, type-safe, trivial migração futura para i18n |
| Observabilidade | @sentry/react + web-vitals | Error tracking com source maps, métricas de performance |
| Testes | Vitest + Testing Library + Playwright + axe-playwright | Unit → component → e2e com a11y automatizado |
| Dev/Doc | Ladle | Storybook alternativo nativo em Vite, desenvolvimento isolado |
| a11y | @axe-core/react (dev) + jest-axe (CI) + NVDA/JAWS manual | Gate automatizado + teste com leitor de tela |
| Bundle Check | size-limit | CI gate para budgets granulares por chunk |
| Font Metrics | @capsizecss/metrics ou fontaine | Cálculo preciso de size-adjust/ascent-override para mitigar CLS |

### 3.2 Tipografia

```css
:root {
  /* Famílias */
  --font-display: 'IBM Plex Sans', system-ui, sans-serif;
  --font-body:    'Literata', 'Source Serif 4', Georgia, serif;
  --font-mono:    'JetBrains Mono', 'Fira Code', monospace;

  /* Pesos semânticos */
  --font-weight-regular:  400;
  --font-weight-medium:   500;
  --font-weight-semibold: 600;

  /* Escala fluida (min: 360px, max: 1280px) */
  --text-xs:   clamp(0.694rem, 0.66rem + 0.17vw, 0.75rem);
  --text-sm:   clamp(0.833rem, 0.79rem + 0.22vw, 0.875rem);
  --text-base: clamp(1rem, 0.95rem + 0.26vw, 1.063rem);
  --text-lg:   clamp(1.125rem, 1.05rem + 0.38vw, 1.25rem);
  --text-xl:   clamp(1.35rem, 1.2rem + 0.75vw, 1.625rem);
  --text-2xl:  clamp(1.62rem, 1.4rem + 1.1vw, 2.1rem);
  --text-3xl:  clamp(1.94rem, 1.6rem + 1.7vw, 2.75rem);

  /* Line heights */
  --leading-tight:  1.2;
  --leading-normal: 1.5;
  --leading-relaxed:1.7;

  /* Letter spacing */
  --tracking-tight:  -0.02em;
  --tracking-normal:  0;
  --tracking-wide:    0.04em;
}
```

**Estratégia de loading crítico (LCP < 2.5s):**
1. **Preload crítico:** Apenas IBM Plex Sans 400/600 (UI chrome). Literata e JetBrains Mono NÃO são preloaded.
2. **Font-display:** `swap` para display e body (Literata). `optional` para JetBrains Mono (lazy load no mount do DocumentViewer).
3. **Mitigação CLS:** Usar `@capsizecss/metrics` para calcular `size-adjust`, `ascent-override`, `descent-override`, `line-gap-override` exatos para Literata→Georgia fallback. Não usar valores arbitrários.
4. **Fallback:** System serif stack (Georgia) renderiza progressivamente com métricas ajustadas para minimizar shift.

**Mapeamento de pesos:**
- Headings: display/600 (semibold)
- Body: body/400 (regular)
- Labels/Captions: display/400 (regular)

**Regras de Stacking Context:**
- **Portalizados (modais, sheets, toasts, dropdowns):** DEVEM usar `React.createPortal` para `document.body` e aplicar tokens `--z-*`.
- **Isolados (AppShell regions):** `isolation: isolate` apenas em regiões estruturais (sidebar, main) para contencimento interno. Componentes dentro dessas regiões que precisam escapar (dropdowns) devem ser portalizados.
- **Proibido:** Valores z-index inline ou tokens em componentes não-portalizados.

### 4.5 BottomSheet (vaul)
```typescript
// vaul v0.9.x API: snapPoints aceita strings ('148px', '50vh', '92dvh') ou números (pixels)
type SnapPoint = `${number}px` | `${number}vh` | `${number}dvh` | number;

interface BottomSheetProps {
  snapPoints?: SnapPoint[];  // ex: ['25vh', '50vh', '92dvh'] ou [200, 400, 800]
  defaultSnap?: number;      // index inicial
  onSnapChange?: (index: number) => void;
  onDismiss?: () => void;
  backdrop?: boolean;
  closeOnBackdrop?: boolean;
  trapFocus?: boolean;
  preventScroll?: boolean;
  handle?: boolean;
  children: ReactNode;
}

// Utilitário obrigatório para converter unidades relativas em pixels (vaul espera pixels no cálculo interno):
function parseSnapPoints(points: SnapPoint[]): number[] {
  return points.map(p => {
    if (typeof p === 'number') return p;
    const match = p.match(/^(\d+(?:\.\d+)?)(px|vh|dvh)$/);
    if (!match) throw new Error(`Invalid snap point: ${p}`);
    const value = parseFloat(match[1]);
    const unit = match[2];
    if (unit === 'px') return value;
    const height = unit === 'dvh' ? window.innerHeight : document.documentElement.clientHeight;
    return (value / 100) * height;
  });
}

// Nota: vaul implementa focus trap, snap points e gestures nativamente (não é baseado em Radix Dialog)
```
