## 1. Real Workflow Value

**Command Palette: genuine win.** Consolidating navigation behind a single shortcut with recent-doc memory and inline results eliminates the tab-hunting workflow that dominates document-heavy tools. Removing the old SearchBar shortcut handler to avoid dual-ownership is the correct call.

**Document Graph: real but narrow.** Surfacing `normative_refs` and `procedure_refs` as a navigable graph gives users structural awareness they previously had to reconstruct mentally. This is a legitimate first step.

**CURRENT_STATE.md: useful today, liability tomorrow.** A hand-maintained ground-truth doc provides onboarding value right now. But the "grounded in real code" claim is only true at the moment of authoring.

## 2. Issues Requiring Immediate Resolution

### CRITICAL — Command Palette shortcut hijacks active input contexts

**Location:** `src/components/CommandPalette/useGlobalShortcut.ts`

The global `Cmd/Ctrl+K` listener fires unconditionally, intercepting the shortcut when users are typing in `<input>`, `<textarea>`, `contentEditable` regions, or during IME composition. This breaks native text editing and is an accessibility violation.

**Required fix:**

```ts
const handleKeyDown = (e: KeyboardEvent) => {
  if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
    const target = e.target as HTMLElement;
    if (
      target.tagName === 'INPUT' ||
      target.tagName === 'TEXTAREA' ||
      target.isContentEditable
    ) {
      return;
    }
    if (e.isComposing) return;
    e.preventDefault();
    callback();
  }
};
```

**Test requirement:** Unit test confirming the handler is a no-op when `document.activeElement` is an input, textarea, contentEditable element, and during IME composition.

### CRITICAL — Shortcut removal creates single point of failure

**Location:** `src/app/Shell.tsx`

Removing the old SearchBar-level `Cmd/Ctrl+K` handler means that if the `CommandPalette` component fails to mount (SSR hydration failure, lazy-load timeout, runtime error boundary catch), keyboard navigation is completely dead with no user-visible indication.

**Required fix:** Register the shortcut listener at the shell level unconditionally. The listener dispatches to the palette if mounted, falls back to focusing the search bar or showing a toast otherwise. The palette does not own the shortcut registration — the shell does.

### CRITICAL — Graph renders unbounded DOM nodes

**Location:** `src/components/DocumentGraph/GraphRenderer.tsx`

DOM-based rendering of graph nodes has no upper bound. Normative datasets with 500+ refs will cause forced reflows exceeding 100ms and memory exhaustion on constrained devices.

**Required fix (ship-blocking):**
- Cap initial render at 50 nodes. Display a count badge ("showing 50 of 347") with an explicit expand action.
- Add a `performance.mark`/`measure` wrapper around render to log render time in dev.
- File a follow-up ticket for canvas-based rendering (e.g., `@antv/g6`, `force-graph`) when node counts routinely exceed 200.

### HIGH — Heuristic relationships displayed as ground truth

**Location:** `src/components/DocumentGraph/DocumentGraph.tsx`

The increment acknowledges that validity/version awareness is heuristic and cautious, but renders all edges identically. Users working in compliance contexts will treat visual relationships as authoritative. This is a trust defect.

**Required fix:**
- Render unverified or heuristic-confidence edges as dashed lines with reduced opacity.
- Add a tooltip on hover: "Relationship inferred heuristically — not verified against source version."
- Add a legend distinguishing verified vs. heuristic edges.

### MEDIUM — CURRENT_STATE.md will drift within one sprint

**Location:** `docs/meta/CURRENT_STATE.md`

A static markdown file claiming to reflect actual code state is a documentation time bomb. The value-to-cost ratio inverts the moment it becomes stale, because stale "ground truth" is worse than no documentation.

**Required fix (next sprint, not ship-blocking):**
- Add a CI check that validates key structural assertions (module existence, export presence) against the source tree. A lightweight script using `find` + `grep` or `ts-morph` is sufficient — full AST generation is premature.
- Add a header to the file: `<!-- Last validated: {CI_TIMESTAMP}. Run 'npm run validate:state' to check. -->`

## 3. Valid Suggestions to Adopt

| Suggestion | Priority | Rationale |
|---|---|---|
| Debounced search with `AbortController` cancellation in `SearchProvider.tsx` | High | Synchronous filtering on large indices will block main thread. Standard async search pattern — no reason to ship without it. |
| ARIA `live` region on command palette results | High | Screen readers are silent on dynamic result updates. This is an accessibility requirement, not a nice-to-have. |
| Debug mode toggle for graph heuristic scores | Low | Useful for development, not user-facing. Implement as a dev-only feature flag. |

## 5. Next Highest-Impact Step

**Ship the four critical/high fixes above**, then:

**Priority 1: Search quality.** The command palette is the new primary navigation surface. Invest in ranked results (fuzzy match scoring, recency weighting, section-aware search). A palette with bad results trains users to ignore it. Debounced async search with cancellation is table stakes here.

**Priority 2: Graph interaction model.** The current graph is view-only. Add click-to-navigate (click a node → open that document) and click-to-focus (click a node → re-center the graph on that node's neighborhood). These two interactions convert the graph from a picture into a tool.

**Priority 3: CI validation for CURRENT_STATE.md.** Do it before the second sprint ends or delete the file. There is no middle ground for documentation that claims to be source-of-truth.
