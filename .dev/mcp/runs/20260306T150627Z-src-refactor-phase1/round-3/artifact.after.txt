Phase 1 Repository Refactor Plan: Safe Migration to src/ Layout

PROPOSED DIRECTION
Create src/backend and src/frontend directories, move implementation packages there with explicit mapping, keep root entrypoint shims for all currently-invoked top-level scripts, update pyproject.toml for src/ layout discovery, and implement validation gates before any code moves. All backend packages move atomically in a single commit to avoid inter-package import breakage during transition.

EXPLICIT PACKAGE MAPPING (Keep-vs-Move Table)
| Package    | Destination        | Rationale                                      | Phase |
|------------|-------------------|------------------------------------------------|-------|
| ingest/    | src/backend/ingest | Core backend data processing                   | 1     |
| search/    | src/backend/search | Backend search/indexing logic                  | 1     |
| dbsync/    | src/backend/dbsync | Database synchronization (backend concern)     | 1     |
| commitment/| src/backend/commitment | Commitment tracking logic (backend)        | 1     |
| web/       | src/frontend/web   | Frontend-facing web server logic               | 1     |
| infra/     | root (defer)       | Shared utilities, audit imports before Phase 2 | 2     |
| deploy/    | root (defer)       | Deployment scripts, defer to Phase 2           | 2     |
| scripts/   | root (defer)       | Utility scripts, audit imports before Phase 2  | 2     |
| tests/     | root (stay)        | Update imports atomically with package moves   | 1     |

Rationale for scripts/: Keep at root in Phase 1 to avoid breaking deployment workflows. Audit scripts/ imports against infra/, deploy/, and backend packages before Phase 1. If scripts/ imports from backend packages, update those imports to use new paths.

ENTRYPOINT HANDLING
| File              | Action                                          | MCP-Safe | Shim Required |
|-------------------|-------------------------------------------------|----------|---------------|
| web_server.py     | Keep at root, shim imports to src/frontend/web | Yes      | Yes           |
| mcp_server.py     | Keep at root, update imports to src/backend    | Yes      | Yes           |
| mcp_es_server.py  | Keep at root, update imports to src/backend    | Yes      | Yes           |
| schema_sync.py    | Keep at root, shim imports to src/backend      | Yes      | Yes           |
| commitment_cli.py | Keep at root, shim imports to src/backend      | Yes      | Yes           |

Rationale: MCP servers are invoked via absolute paths in Claude Desktop, VS Code, and other MCP client manifests. All entrypoints require sys.path shims to discover src/ packages while remaining at root. Shims must add both src/ and root to sys.path to support deferred packages (infra/, deploy/).

ENTRYPOINT SHIM MECHANISM (Concrete Implementation for All Five Entrypoints)

PRE-MIGRATION VALIDATION (Blocking Gate):
Before creating shims, verify target module structure exists:
- Confirm web/main.py exists with callable main() function
- Confirm search/mcp.py exists with callable main() function
- Confirm search/mcp_es.py exists with callable main() function
- Confirm dbsync/sync.py exists with callable main() function
- Confirm commitment/cli.py exists with callable main() function
- Extract any module-level initialization code (logging, env vars, signal handlers) from current entrypoints into target modules before shim replacement

web_server.py shim (template for all five):
```python
#!/usr/bin/env python3
import sys
from pathlib import Path

_root = Path(__file__).parent.resolve()
_src = _root / "src"

# Support both src/ packages and root-level deferred packages (infra/, deploy/, scripts/)
for p in [str(_src), str(_root)]:
    if p not in sys.path:
        sys.path.insert(0, p)

# Fail fast if src/ directory is missing
if not _src.exists():
    print(f"FATAL: src/ directory not found at {_src}", file=sys.stderr)
    sys.exit(1)

try:
    from frontend.web.main import main as _main
except ImportError as e:
    print(f"FATAL: Cannot import frontend.web.main: {e}", file=sys.stderr)
    print(f"Script dir: {_root}, sys.path: {sys.path}", file=sys.stderr)
    sys.exit(1)

if __name__ == "__main__":
    _main()
```

Key requirements:
- Shims manipulate sys.path explicitly before importing
- Shims add both src/ and root to sys.path to support deferred packages
- Shims are NOT named identically to modules they import (avoid circular import errors)
- All five entrypoints use consistent pattern with try/except error handling
- Test each shim independently before merging
- Error handling logs clear messages with sys.path and script directory on failure
- _main alias used to avoid shadowing potential local main definition in shim

IMPORT PATH STRATEGY
1. Pre-move: Use AST-based refactoring tool (libcst, rope, or ast-grep with Python grammar) to rewrite all internal imports. DO NOT use regex-based refactoring.
   - Convert absolute imports: `from ingest import X` → `from backend.ingest import X`
   - Convert absolute imports: `import ingest` → `import backend.ingest`
   - Convert absolute imports: `from search.module import X` → `from backend.search.module import X`
   - Convert absolute imports: `import search as s` → `import backend.search as s`
   - DO NOT change relative imports within packages (e.g., `from .module` stays `from .module`)
   - Handle star imports: `from ingest import *` → `from backend.ingest import *`
2. Update pyproject.toml for src/ layout:
```toml
[tool.setuptools.packages.find]
where = ["src"]
# Remove include filter to auto-discover all packages under src/
# OR use inclusive globs: include = ["backend*", "frontend*"]
```
3. Add src/backend/__init__.py and src/frontend/__init__.py with explicit __all__ exports for backward-compatible imports during transition
4. Note: sys.path manipulation in shims is temporary and will be removed in Phase 2 when entrypoints move to src/

PYPROJECT.TOML UPDATES REQUIRED

MIGRATION ORDER (Atomic, Not Incremental)
1. Create pre-move branch: git checkout -b pre-phase1-refactor
2. Create src/backend and src/frontend empty dirs with __init__.py
3. Update pyproject.toml and pytest configuration
4. Run import audit gates (infra/deploy dependencies, MCP server imports, dynamic imports, scripts/ dependencies)
5. Run AST-based refactoring on ALL backend packages simultaneously (ingest, search, dbsync, commitment)
6. Run AST-based refactoring on frontend package (web)
7. Run AST-based refactoring on tests/ directory with same patterns
8. Update all five entrypoint shims atomically
9. Clear __pycache__: find . -name "*.pyc" -delete
10. git mv all packages to src/ locations in single commit (update imports and move files in same commit)
11. Run tests, verify no regressions
12. Verify git mv rename detection: git diff --stat HEAD~1 | grep "R100"
13. Final validation: all entrypoints, MCP servers, CI/CD, type checking, docs
14. Tag pre-refactor state: git tag pre-phase1-refactor <commit-hash>

RISK REGISTER
| Risk                              | Severity | Mitigation                                    |
|-----------------------------------|----------|-----------------------------------------------|
| ImportError cascade during transition | High   | Atomic migration of all backend packages in single commit |
| CI/CD pipeline breakage           | High     | Update CI configs in same PR, test on branch   |
| IDE autocomplete failures         | Medium   | Document new paths, update IDE workspace settings |
| Third-party tool breakage (linters, formatters) | Medium | Update tool configs (pyproject.toml, .pre-commit-config.yaml) |
| MCP integration breakage          | Critical | All entrypoints get sys.path shims with root+src paths, atomic config updates, test from arbitrary CWD |
| tests/ directory import breakage  | High     | Update test imports atomically with package moves |
| infra/deploy cross-tree dependencies | Critical | Block Phase 1 if any backend/MCP code imports infra/; update shims to include root in sys.path; move infra/ to src/backend/infra immediately if dependencies found |
| scripts/ cross-tree dependencies  | High     | Audit scripts/ imports before Phase 1; update to new paths or defer scripts/ to Phase 2 |
| Long-running feature branch divergence | Medium | Complete Phase 1 in single day, rebase frequently |
| Python version compatibility      | Medium   | Verify src/ layout works on minimum supported Python version (3.3+) |
| Dynamic imports bypassing analysis | Medium  | Scan for importlib/__import__/import_string usage before move |
| Cached bytecode (.pyc) conflicts  | Medium   | Run find . -name "*.pyc" -delete before testing on feature branch |
| Existing venv cached .pth files   | Medium   | Document venv reinstallation requirement; verify pip install -e . works in fresh venv |

ROLLBACK PLAN
If Phase 1 causes unacceptable breakage:
1. Do not merge to main; stay on feature branch
2. If already merged: checkout pre-phase1-refactor branch instead of git revert
3. Canonical rollback is checkout of pre-phase1-refactor branch. Git tracks all changes.
4. Restore original pyproject.toml configuration from pre-move branch
5. Document failure mode and adjust plan before retry
6. Note: git revert may not cleanly restore import changes across many files; branch checkout is preferred

DEFERRAL DECISIONS (Phase 2)
- infra/, deploy/, and scripts/ remain at root in Phase 1
- Before Phase 2: Audit all imports from infra/ into ingest/search/dbsync/commitment/web
- If cross-tree dependencies exist, either move infra/ to src/backend/infra or create compatibility shims
- Explicitly audit whether mcp_server.py and mcp_es_server.py have dependencies on infra/ or deploy/; if they do, those cannot be deferred to Phase 2 without breaking the MCP servers
- Phase 2 decision criteria: infra/ moves to src/backend/infra only if zero entrypoints depend on it after Phase 1
- Update CI/CD files that reference old paths: .github/workflows/, Makefile, tox.ini, Dockerfile
