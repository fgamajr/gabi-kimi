# Benchmark Normalization Plan: Field Coverage Fix at Adapter Layer

## 1. Root Cause Analysis (Required Before Any Normalization)

**Phase 0: Field Coverage Audit**

| Check | Action | Tool/Command |
|-------|--------|--------------|
| PG table schema | Verify `issuing_organ` and `body_plain` columns exist in the source table | `\d documents` in PG |
| PGSearchAdapter SQL | Review current SELECT clause in adapter | `grep -n "SELECT" src/adapters/pg_search.py` |
| ES index mapping | Verify fields are indexed and returned | `GET /documents/_mapping` |
| Grader field requirements | Extract full list of required fields from grader code | Review `src/grader/*.py` |

**Decision Matrix:**
- If PG schema lacks field → schema migration required before benchmark
- If PG adapter omits field in SELECT → fix adapter projection
- If ES index missing field → reindex required before benchmark
- If adapter can project field → fix at adapter, NOT via post-retrieval hydration

## 2. Canonical SearchResult Schema Contract

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "SearchResult",
  "type": "object",
  "required": ["id", "issuing_organ", "body_plain", "score", "rank"],
  "properties": {
    "id": { "type": "string" },
    "issuing_organ": { "type": ["string", "null"] },
    "body_plain": { "type": ["string", "null"] },
    "score": { "type": "number" },
    "rank": { "type": "integer" },
    "metadata": { "type": "object" }
  }
}
```

All adapters MUST return objects conforming to this schema. Missing keys are benchmark violations; null values are permitted for missing data.

## 3. Implementation Plan (Adapter-Level Fix, NOT Post-Retrieval Hydration)

### 3.1 PGSearchAdapter Fix

**File: `src/adapters/pg_search.py`**

```python
# BEFORE (omits fields)
SELECT id, ts_rank(to_tsvector(content), plainto_tsquery($1)) AS score
FROM documents WHERE ...

# AFTER (project required fields)
SELECT 
    id, 
    issuing_organ, 
    body_plain,
    ts_rank(to_tsvector(content), plainto_tsquery($1)) AS score
FROM documents WHERE ...
```

If `issuing_organ` or `body_plain` reside in a related table, use LEFT JOIN:

```sql
SELECT d.id, org.name AS issuing_organ, d.body_plain, ...
FROM documents d
LEFT JOIN organizations org ON d.organ_id = org.id
WHERE ... 
```

### 3.2 Validation Layer (Pre-Grader)

**File: `src/benchmark/schema_validator.py`**

```python
from jsonschema import validate, ValidationError

SCHEMA = {...}  # See Section 2

def validate_results(results: list[dict], backend: str) -> None:
    """
    Fails fast if any result missing required keys.
    Null values allowed; omitted keys prohibited.
    """
    for i, doc in enumerate(results):
        try:
            validate(instance=doc, schema=SCHEMA)
        except ValidationError as e:
            raise BenchmarkError(
                f"Backend {backend} returned malformed result at index {i}: "
                f"missing key '{e.validator_value}' (null allowed, omitted prohibited)"
            )
```

**Integration in harness:**

```python
# src/benchmark/harness.py
def run_benchmark(query: str, backend: str):
    results = adapter.execute(query)
    validate_results(results, backend)  # Fails fast before grading
    score = grader.evaluate(query, results)
    return score
```

### 3.3 Performance Impact Assessment

- **Action:** Profile PG query latency with and without added columns
- **Threshold:** If latency increase > 15% or > 5ms p95, escalate to team lead
- **Contingency:** If body_plain is too large, negotiate with grader team to use `LEFT(body_plain, 500)` or accept null with documented limitation

## 4. Severity-Ranked Concerns

| Severity | Concern | Mitigation |
|----------|---------|------------|
| **Critical** | Post-retrieval hydration masks production defect | BANNED. Fix at adapter level only |
| **Critical** | Benchmark runs with schema violations | Validation layer fails fast |
| **High** | PG lacks columns in schema | Halt benchmark; require schema migration |
| **High** | ES index missing required fields | Halt benchmark; require reindex |
| **Medium** | Performance regression > 15% | Assess; negotiate smaller field or accept limitation |
| **Low** | Null values cause grader issues | Grader should treat null as "no match" |

## 6. Testing Strategy

```python
# tests/test_adapter_schema.py
def test_pg_adapter_returns_issuing_organ():
    results = pg_adapter.search("test query")
    assert all("issuing_organ" in doc for doc in results)

def test_schema_validator_rejects_omitted_key():
    bad_results = [{"id": "1", "score": 0.5}]  # missing issuing_organ
    with raises(BenchmarkError, match="missing key"):
        validate_results(bad_results, "pg")

def test_null_values_permitted():
    results = [{"id": "1", "issuing_organ": None, ...}]
    validate_results(results, "pg")  # Should pass
```
