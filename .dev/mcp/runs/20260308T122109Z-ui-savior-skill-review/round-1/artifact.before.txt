**Hero Sections (landing.love)**

**Feature Sections (saaspo.com)**

- Visual evidence: screenshots, illustrations, or live demos

**CTA Sections (cta.gallery)**
- Pattern: contrast band / floating bar / inline form / modal trigger
- Copy formula: [Action verb] + [Benefit] + [Timeframe/ease]
- Button hierarchy: primary (filled) + secondary (outline/ghost)

**Cards & Lists (curated.design)**
- Pattern: media-top / horizontal / overlay / minimal
- Hover states: lift, border glow, scale, content reveal

#### Motion & Animation (appmotion.design)
- **Entrance animations**: fade-up, slide-in, scale-in (use `IntersectionObserver`)
- **Micro-interactions**: button press feedback, toggle animations, form field focus
- **Page transitions**: crossfade, slide, morph (via `View Transitions API` or `framer-motion`)
- **Loading states**: skeleton screens, shimmer, progressive blur
- **Performance rules**:
  - Only animate `transform` and `opacity` (GPU-composited)
  - Use `will-change` sparingly, only on elements about to animate
  - Respect `prefers-reduced-motion` ŌĆö always provide fallback
  - Keep durations 150ŌĆō400ms (micro: 150ms, reveals: 300ms, transitions: 400ms)

#### Responsive Strategy
- Mobile-first breakpoints: `sm:640` `md:768` `lg:1024` `xl:1280`
- Touch targets: minimum 44x44px
- Typography scale: fluid `clamp()` or Tailwind responsive prefixes
- Layout shifts: stack on mobile, grid on desktop
- Navigation: drawer/bottom-sheet on mobile, full nav on desktop

## Design Principles (Always Apply)

1. **Contrast creates hierarchy** ŌĆö Use size, weight, and color to guide the eye
2. **Whitespace is a feature** ŌĆö Generous padding signals quality; cramped layouts signal amateur
3. **One primary action per view** ŌĆö Every screen has ONE thing you want users to do
4. **Motion with purpose** ŌĆö Animate to guide attention, confirm actions, or show relationships
5. **Mobile is the real design** ŌĆö Desktop is the easy version; start with constraints
6. **Consistency over creativity** ŌĆö Reuse patterns within a product; novelty per-component is noise
7. **Performance is UX** ŌĆö A fast, simple design beats a slow, beautiful one

## Anti-Patterns (Never Recommend)

- Carousels for critical content (low engagement past slide 1)
- Hamburger menus on desktop (hides navigation)
- Autoplay video/audio (hostile UX)
- Light gray text on white backgrounds (accessibility fail)
- More than 2 fonts (visual noise)
- Animations longer than 500ms (feels sluggish)
- Custom scrollbars (break platform expectations)
- Infinite scroll without "back to top" and URL state
