# Production Frontend Implementation Spec v3

## 0. Repo Context & Constraints

- **Current state**: Frontend served as static HTML/Alpine/Tailwind from `src/frontend/web/` via `src/backend/apps/web_server.py`
- **Backend endpoints preserved**: `/api/search`, `/api/suggest`, `/api/autocomplete`, `/api/top-searches`, `/api/search-examples`, `/api/document/{doc_id}`
- **Stack**: React 18 functional components/hooks, Tailwind CSS core utilities, pure CSS animations, SVG icons inlined as components, CSS custom properties
- **Breakpoints**: 375px (mobile-first) в†’ 768px (tablet) в†’ 1440px (desktop)
- **Aesthetic**: Premium/confident Nubank + Flighty. Dark-mode-first. No Material/Bootstrap/shadcn defaults, no decorative chrome, no spinners, no ease/linear timing.

**vite.config.js**:
- `build.outDir`: `dist`
- `build.assetsDir`: `assets`
- `base`: `/` (set to `/v2/` during migration phase, revert to `/` after cutover)
- Output: hashed JS chunks, hashed CSS, index.html
- `rollup-plugin-visualizer` included for bundle analysis (html report, not shipped)

**package.json scripts**:
- `dev`: `vite` (local dev server with HMR, proxies `/api/*` to FastAPI)
- `build`: `vite build` (outputs to `dist/`)
- `build:analyze`: `ANALYZE=true vite build` (generates bundle report)
- `preview`: `vite preview`

No runtime transpilation. No CDN dependencies. All dependencies vendored into the bundle at build time.

## 2. web_server.py Changes

Minimal, surgical changes to existing server. **Route registration order is critical**: all API routes first, then static mounts, then SPA catch-all last.

### 2.1 Production Serving

```python
# 1. API routes registered FIRST (existing вЂ” no changes)
# app.include_router(api_router, prefix="/api")

# 2. Static assets mount SECOND (hashed filenames = immutable cache)
app.mount(
    "/assets",
    StaticFiles(directory="src/frontend/web/dist/assets"),
    name="assets"
)

# 3. SPA catch-all registered LAST
@app.get("/{path:path}")
async def spa_fallback(path: str):
    # Safety: never serve HTML for API paths
    if path.startswith("api/") or path.startswith("assets/"):
        raise HTTPException(status_code=404)
    return FileResponse(
        "src/frontend/web/dist/index.html",
        headers={"Cache-Control": "no-cache"}
    )

# Root handler (empty path doesn't match {path:path})
@app.get("/")
async def spa_root():
    return FileResponse(
        "src/frontend/web/dist/index.html",
        headers={"Cache-Control": "no-cache"}
    )
```

**Cache headers** (set via StaticFiles middleware or custom middleware):
- `dist/assets/*` (hashed filenames): `Cache-Control: public, max-age=31536000, immutable`
- `dist/index.html`: `Cache-Control: no-cache` (always revalidated)

### 2.2 Migration Strategy

**Phase 1 вЂ” Parallel deploy**: Serve React app at `/v2/` path prefix. Vite builds with `base: '/v2/'`. Legacy Alpine app continues at `/`. Backend adds:
```python
app.mount("/v2/assets", StaticFiles(directory="src/frontend/web/dist/assets"), name="v2-assets")

@app.get("/v2/{path:path}")
async def spa_v2_fallback(path: str):
    return FileResponse("src/frontend/web/dist/index.html")
```

**Phase 2 вЂ” Parity confirmed**: Swap `/` to serve React (`base: '/'`), move legacy Alpine files to `src/frontend/web/_legacy/`. Legacy remains accessible at `/_legacy/` for 2 weeks.

**Phase 3 вЂ” Cleanup**: Remove `_legacy/` directory, remove `/v2` routes, remove legacy mount.

### 2.3 Rollback Plan

If React deploy fails after Phase 2 cutover:
1. Revert `web_server.py` to serve `_legacy/` at `/` (single config change)
2. `_legacy/` files are still present for 2-week window вЂ” no rebuild required
3. Rollback is a server restart, not a deploy

### 2.4 Smoke Tests

After any deploy phase, run:
```bash
# API routes still return JSON
curl -s -o /dev/null -w "%{content_type}" localhost:8000/api/search?q=test | grep application/json
# SPA serves HTML
curl -s -o /dev/null -w "%{content_type}" localhost:8000/ | grep text/html
# Assets serve with cache headers
curl -sI localhost:8000/assets/main.abc123.js | grep "immutable"
# Unknown paths serve SPA (not 404)
curl -s -o /dev/null -w "%{http_code}" localhost:8000/document/xyz | grep 200
```

### 3.7 Theme FOUC Prevention

Inline script in `index.html <head>`, executed synchronously before any paint:

```html
<script>
  (function(){
    var t = localStorage.getItem('theme');
    if (!t) t = window.matchMedia('(prefers-color-scheme:light)').matches ? 'light' : 'dark';
    document.documentElement.setAttribute('data-theme', t);
    document.querySelector('meta[name="theme-color"]').content = t === 'dark' ? '#0A0A0B' : '#FFFFFF';
  })();
</script>
<meta name="theme-color" content="#0A0A0B">
```

This script is <300 bytes minified. It runs before React hydration, eliminating theme flash entirely.

## 4. Motion System (`src/styles/animations.css`)

**Non-negotiable rules**:
- Only `transform` and `opacity` animated (compositor-only, no layout thrash)
- No `ease` or `linear` timing вЂ” all custom cubic-bezier curves
- All animations gated by `prefers-reduced-motion`
- `will-change` applied only during animation, removed after completion via `animationend` listener

### 4.1 Motion Tokens

```css
:root {
  --duration-instant: 0ms;
  --duration-fast: 120ms;
  --duration-normal: 200ms;
  --duration-slow: 350ms;
  --duration-glacial: 500ms;

  /* Deceleration вЂ” element entering/settling */
  --ease-out: cubic-bezier(0.2, 0, 0, 1);
  /* Acceleration вЂ” element exiting */
  --ease-in: cubic-bezier(0.4, 0, 1, 0.8);
  /* Standard вЂ” repositioning/morphing */
  --ease-standard: cubic-bezier(0.2, 0, 0.2, 1);
  /* Emphasis вЂ” springy/attention */
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);

  --stagger-delay: 40ms;
  --stagger-max: 400ms;  /* cap: stagger never exceeds this total */
}

@media (prefers-reduced-motion: reduce) {
  :root {
    --duration-fast: 0ms;
    --duration-normal: 0ms;
    --duration-slow: 0ms;
    --duration-glacial: 0ms;
    --stagger-delay: 0ms;
    --stagger-max: 0ms;
  }
}
```

### 4.2 Reduced Motion Behavior

| Animation Category | Reduced Motion Behavior |
|--------------------|------------------------|
| Screen transitions (enter/exit) | Instant state change (0ms), no transform |
| List stagger | Instant, all items appear simultaneously |
| Card scale-in | Instant, appears at final scale |
| Bottom sheet slide | Instant, appears at final position |
| Focus ring | **Preserved** вЂ” visual feedback maintained, transition to 0ms |
| Skeleton pulse | **Replaced** вЂ” static 0.5 opacity instead of pulsing |
| Theme toggle | Instant вЂ” no cross-fade |

### 4.5 `useStaggerDelay` Hook

```javascript
/**
 * Returns animationDelay in ms for staggered list animations.
 * Caps total stagger at --stagger-max to prevent long cascades.
 * @param {number} index - Item index in the list
 * @param {number} [total] - Total number of items (for adaptive delay)
 * @returns {string} CSS delay value, e.g. "80ms"
 */
function useStaggerDelay(index, total) {
  const DELAY = 40;  // matches --stagger-delay
  const MAX = 400;   // matches --stagger-max
  const effectiveDelay = total && total > 0
    ? Math.min(DELAY, MAX / total)
    : DELAY;
  return `${Math.min(index * effectiveDelay, MAX)}ms`;
}
```
