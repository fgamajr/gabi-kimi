# Production Frontend Implementation Spec v6

## 0. Repo Context & Constraints

- **Current state**: Frontend served as static HTML/Alpine/Tailwind from `src/frontend/web/` via `src/backend/apps/web_server.py`
- **Backend endpoints preserved**: `/api/search`, `/api/suggest`, `/api/autocomplete`, `/api/top-searches`, `/api/search-examples`, `/api/document/{doc_id}`
- **Stack**: React 18 functional components/hooks, Tailwind CSS core utilities, pure CSS animations, SVG icons inlined as components, CSS custom properties
- **Breakpoints**: 375px (mobile-first) → 768px (tablet) → 1440px (desktop)
- **Aesthetic**: Premium/confident Nubank + Flighty. Dark-mode-first. No Material/Bootstrap/shadcn defaults, no decorative chrome, no spinners, no ease/linear timing.
- **Node requirement**: >=18.0.0 (documented in `engines` field)
- **API contract**: All URLs in API responses are relative paths. No absolute URLs returned by any endpoint.
- **Dependency policy**: Any new runtime dependency requires a PR with bundle-size-before/after from `build:analyze` and approval from 2 reviewers.

## 1. Routing Strategy

**Decision: History-based routing with wouter.**

Clean URLs (`/search`, `/document/abc123`) for premium UX, shareable links, and professional appearance. This requires the SPA catch-all in Section 2.

- **Library**: `wouter` v3 — 1.5KB gzipped, history-based by default, no hash routing.
- **Route definitions** live in `app/routes.js` as a flat map of path → lazy component.
- **No nested routing** — wouter doesn't support it natively and we don't need it.
- **ActionSheet** is rendered as an overlay within DocumentScreen, driven by URL query parameter `?action=edit` (see Section 5.3).

**Search URL contract**: `/search?q=encoded_query`. SearchScreen reads initial query from `window.location.search` on mount. Typing updates URL via `history.replaceState` (no history pollution). Submitting or selecting a suggestion uses `history.pushState` so back-button returns to the previous query. The SPA fallback in `web_server.py` serves `index.html` regardless of query string — the browser preserves query params client-side.

### 1.6 Navigation Flow Specification

**Spatial model**: Stack-based navigation. Screens exist on a conceptual left-to-right axis ordered: Dashboard(0) → Search(1) → Document(2).

**Direction detection**: `useNavigationDirection` tracks a monotonically incrementing position counter stored in `history.state.navPos`. Every `pushState` increments the counter. When `popstate` fires, the new position is compared to the last known position — lower means back, higher means forward. This is immune to the non-linear history traversal bug (Dashboard→Search→Document→Search shows correct "back" animation because the position decremented).

```javascript
// shared/hooks/useNavigationDirection.js
import { useRef, useEffect, useCallback } from 'react';

let navPos = history.state?.navPos ?? 0;

export function useNavigationDirection() {
  const directionRef = useRef('forward');
  const lastPosRef = useRef(navPos);

  useEffect(() => {
    function onPopState(e) {
      const newPos = e.state?.navPos ?? 0;
      directionRef.current = newPos < lastPosRef.current ? 'back' : 'forward';
      lastPosRef.current = newPos;
    }
    window.addEventListener('popstate', onPopState);
    return () => window.removeEventListener('popstate', onPopState);
  }, []);

  const navigate = useCallback((url) => {
    navPos += 1;
    history.pushState({ navPos }, '', url);
    lastPosRef.current = navPos;
    directionRef.current = 'forward';
  }, []);

  return { directionRef, navigate };
}
```

**Transition rules**:
- Forward navigation: current screen exits left + fades, new screen enters from right + fades in
- Back navigation: current screen exits right + fades, new screen enters from left + fades in
- Same-screen re-render (e.g., Document → Document): crossfade only, no slide
- ActionSheet: slides up from bottom (overlay, not a route transition)

**Focus management on route change**: `AnimatedRoute` calls `mainContentRef.current.focus()` after the route transition completes, where `mainContentRef` points to `<main id="main-content" tabIndex="-1">`. The `tabIndex="-1"` makes it programmatically focusable without adding it to the tab order. This satisfies WCAG 2.4.3 (Focus Order).

**Route announcements**: The `#route-announcer` div (`aria-live="assertive" role="status"`) receives the new screen name (e.g., "Navigated to Document Details") after each route change. This is separate from focus management — screen readers announce the navigation while focus moves to the main content area.

**Browser back button**: Triggers identical animation to in-app back navigation via the `popstate`-based direction detection.

**Deep linking**: `/document/abc123` opens DocumentScreen with ActionSheet closed. `/document/abc123?action=edit` opens DocumentScreen with ActionSheet open. Back button from open ActionSheet closes it (pops the `?action=` state) without leaving the document.

**Document title updates**: `useDocumentTitle` hook called in each screen component:
```javascript
// shared/hooks/useDocumentTitle.js
import { useEffect } from 'react';

export function useDocumentTitle(title) {
  useEffect(() => {
    const prev = document.title;
    document.title = title ? `${title} — BuscaLeg` : 'BuscaLeg';
    return () => { document.title = prev; };
  }, [title]);
}
```

**scripts/check-bundle-size.js**:

```javascript
import { readdirSync, statSync, readFileSync, existsSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';
import { gzipSync } from 'node:zlib';

const __dirname = dirname(fileURLToPath(import.meta.url));
const DIST = join(__dirname, '..', 'dist', 'assets');

// Level 6 matches real-world CDN/browser compression
const GZIP_LEVEL = 6;

const JS_BUDGETS = {
  'vendor': { max: 50_000, label: 'Vendor chunk' },
  'index':  { max: 40_000, label: 'Initial app chunk' },
};
const JS_TOTAL_MAX = 170_000;
const CSS_TOTAL_MAX = 20_000;
const HASH_PATTERN = /-[a-zA-Z0-9]{8,}\./;

if (!existsSync(DIST)) {
  console.error('FAIL: dist/assets not found. Run `npm run build` first.');
  process.exit(1);
}

let jsTotal = 0;
let cssTotal = 0;
let failed = false;

for (const file of readdirSync(DIST)) {
  const filePath = join(DIST, file);
  const raw = statSync(filePath).size;
  const contents = readFileSync(filePath);
  const gzipped = gzipSync(contents, { level: GZIP_LEVEL }).length;

  // Verify content-hash in filename
  if (!HASH_PATTERN.test(file)) {
    console.error(`FAIL: ${file} does not contain a content hash. Immutable caching is unsafe.`);
    failed = true;
  }

  if (file.endsWith('.js')) {
    jsTotal += gzipped;
    for (const [key, budget] of Object.entries(JS_BUDGETS)) {
      if (file.includes(key) && gzipped > budget.max) {
        console.error(
          `FAIL: ${budget.label} (${file}) is ${gzipped}B gzipped, budget ${budget.max}B`
        );
        failed = true;
      }
    }
    console.log(`  JS  ${file}: ${raw}B raw, ${gzipped}B gzipped`);
  } else if (file.endsWith('.css')) {
    cssTotal += gzipped;
    console.log(`  CSS ${file}: ${raw}B raw, ${gzipped}B gzipped`);
  }
}

if (jsTotal > JS_TOTAL_MAX) {
  console.error(`FAIL: Total JS ${jsTotal}B gzipped exceeds budget ${JS_TOTAL_MAX}B`);
  failed = true;
}
if (cssTotal > CSS_TOTAL_MAX) {
  console.error(`FAIL: Total CSS ${cssTotal}B gzipped exceeds budget ${CSS_TOTAL_MAX}B`);
  failed = true;
}

console.log(`\nTotals: JS ${jsTotal}B, CSS ${cssTotal}B (gzipped, level ${GZIP_LEVEL})`);
process.exit(failed ? 1 : 0);
```

### 2.1 Production Serving

```python

            "Run 'cd src/frontend/web && npm run build' first."

# To:     app = FastAPI(..., lifespan=lifespan)

# --- ADD: Pure ASGI middleware for immutable asset caching ---
# (avoids BaseHTTPMiddleware thread-wrapping overhead)
class ImmutableAssetsCacheMiddleware:
    def __init__(self, app: ASGIApp):
        self.app = app

    async def __call__(self, scope: Scope, receive: Receive, send: Send):
        if scope["type"] == "http" and scope["path"].startswith("/assets/"):

                if message["type"] == "http.response.start":
                    headers = list(message.get("headers", []))
                    headers.append(
                        (b"cache-control", b"public, max-age=31536000, immutable")
                    )
                    message = {**message, "headers": headers}
                await send(message)
            await self.app(scope, receive, send_with_cache)
        else:
            await self.app(scope, receive, send)

app.add_middleware(ImmutableAssetsCacheMiddleware)

@app.exception_handler(StarletteHTTPException)
async def spa_or_api_exception_handler(request: Request, exc: StarletteHTTPException):
    # Non-404 errors: preserve original status code and detail
    if exc.status_code != 404:
        return JSONResponse(
            {"detail": exc.detail or "Server error"},
            status_code=exc.status_code,
        )
    # API and infrastructure paths: always return JSON 404
    if request.url.path.startswith(("/api/", "/healthz")):
        return JSONResponse(
            {"detail": exc.detail or "Not found"},
            status_code=404,
        )
    # Browser navigation to unknown paths: serve SPA
    accept = request.headers.get("accept", "")
    if "text/html" in accept:
        return FileResponse(
            str(DIST_INDEX),
            headers={"Cache-Control": "no-cache"},
            media_type="text/html",
        )
    # Non-browser, non-API 404: JSON fallback
    return JSONResponse(
        {"detail": exc.detail or "Not found"},
        status_code=404,
    )
```

**Key design decisions**:
- `/healthz` is a dedicated endpoint returning JSON — never masked by SPA fallback
- 404 handler ONLY intercepts 404s; all other status codes (403, 500, etc.) are passed through unchanged
- Path prefix guard: `/api/*` and `/healthz` paths always return JSON 404s regardless of `Accept` header
- Pure ASGI middleware for caching avoids `BaseHTTPMiddleware` thread-wrapping overhead
- `index.html` always gets `Cache-Control: no-cache` (browser revalidates on every navigation)
- `validate_frontend_build()` is wired into FastAPI's `lifespan` context manager — crashes on boot if `dist/` is missing
