# Production Frontend Implementation Spec v5

## 0. Repo Context & Constraints

- **Current state**: Frontend served as static HTML/Alpine/Tailwind from `src/frontend/web/` via `src/backend/apps/web_server.py`
- **Backend endpoints preserved**: `/api/search`, `/api/suggest`, `/api/autocomplete`, `/api/top-searches`, `/api/search-examples`, `/api/document/{doc_id}`
- **Stack**: React 18 functional components/hooks, Tailwind CSS core utilities, pure CSS animations, SVG icons inlined as components, CSS custom properties
- **Breakpoints**: 375px (mobile-first) в†’ 768px (tablet) в†’ 1440px (desktop)
- **Aesthetic**: Premium/confident Nubank + Flighty. Dark-mode-first. No Material/Bootstrap/shadcn defaults, no decorative chrome, no spinners, no ease/linear timing.
- **Node requirement**: >=18.0.0 (documented in `engines` field)
- **API contract**: All URLs in API responses are relative paths. No absolute URLs returned by any endpoint.
- **Dependency policy**: Any new runtime dependency requires a PR with bundle-size-before/after from `build:analyze` and approval from 2 reviewers.

---

## 1. Routing Strategy

**Decision: History-based routing with wouter.**

Clean URLs (`/search`, `/document/abc123`) for premium UX, shareable links, and professional appearance. This requires the SPA catch-all in Section 2.

- **Library**: `wouter` v3 вЂ” 1.5KB gzipped, history-based by default, no hash routing.
- **Route definitions** live in `app/routes.js` as a flat map of path в†’ lazy component.
- **No nested routing** вЂ” wouter doesn't support it natively and we don't need it. ActionSheet is rendered as an overlay within DocumentScreen (not a child route вЂ” state-only, not URL-routable; see Section 5.3).

### 1.6 Navigation Flow Specification

**Spatial model**: Stack-based navigation. Screens exist on a conceptual left-to-right axis ordered: Dashboard(0) в†’ Search(1) в†’ Document(2).

**Transition rules**:
- Navigating to a higher-index screen: current screen slides left + fades, new screen slides in from right + fades in
- Navigating to a lower-index screen (back): current screen slides right + fades, new screen slides in from left + fades in
- Same-index navigation (e.g., Document в†’ Document): crossfade only, no slide
- ActionSheet: slides up from bottom (not a route transition)

**Browser back button**: Triggers identical animation to in-app back navigation вЂ” wouter's history integration fires the same location change, and `useNavigationDirection` detects the index decrease.

**Deep linking**: ActionSheet is NOT URL-routable. Direct link to `/document/abc123` opens DocumentScreen with ActionSheet closed. This avoids complexity of encoding overlay state in URL and prevents broken states from stale links.

**Document title updates**: `useDocumentTitle` hook called in each screen component:
```javascript
// shared/hooks/useDocumentTitle.js
import { useEffect } from 'react';

export function useDocumentTitle(title) {
  useEffect(() => {
    const prev = document.title;
    document.title = title ? `${title} вЂ” App` : 'App';
    return () => { document.title = prev; };
  }, [title]);
}
```

**Route announcements for screen readers**: `AnimatedRoute` renders a visually-hidden `<div aria-live="assertive" role="status">` that announces the new screen name on route change (e.g., "Navigated to Document Details").

**scripts/check-bundle-size.js**:

```javascript
import { readdirSync, statSync, readFileSync, existsSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';
import { gzipSync } from 'node:zlib';

const __dirname = dirname(fileURLToPath(import.meta.url));
const DIST = join(__dirname, '..', 'dist', 'assets');
const BUDGETS = {
  'vendor': { max: 45_000, label: 'Vendor chunk' },
  'index':  { max: 40_000, label: 'Initial app chunk' },
};
const TOTAL_MAX = 160_000;

if (!existsSync(DIST)) {
  console.error('FAIL: dist/assets not found. Run `npm run build` first.');
  process.exit(1);
}

let total = 0;
let failed = false;

for (const file of readdirSync(DIST)) {
  if (!file.endsWith('.js')) continue;
  const filePath = join(DIST, file);
  const raw = statSync(filePath).size;
  const contents = readFileSync(filePath);
  const gzipped = gzipSync(contents, { level: 9 }).length;
  total += gzipped;

  for (const [key, budget] of Object.entries(BUDGETS)) {
    if (file.includes(key) && gzipped > budget.max) {
      console.error(
        `FAIL: ${budget.label} (${file}) is ${gzipped}B gzipped, budget ${budget.max}B`
      );
      failed = true;
    }
  }
  console.log(`  ${file}: ${raw}B raw, ${gzipped}B gzipped`);
}

if (total > TOTAL_MAX) {
  console.error(`FAIL: Total JS ${total}B gzipped exceeds budget ${TOTAL_MAX}B`);
  failed = true;
}

process.exit(failed ? 1 : 0);
```

## 2. web_server.py Changes

Minimal, surgical changes to existing server. Shown as additions to the existing file вЂ” **do not replace existing API router registration**.

### 2.1 Production Serving

```python
# --- ADD these imports at top of existing web_server.py ---
import os
from pathlib import Path
from contextlib import asynccontextmanager
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse, JSONResponse
from starlette.exceptions import HTTPException as StarletteHTTPException
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

DIST_DIR = Path(__file__).resolve().parent.parent.parent / "frontend" / "web" / "dist"
DIST_INDEX = DIST_DIR / "index.html"
DIST_ASSETS = DIST_DIR / "assets"

# --- ADD: Startup guard ---
def validate_frontend_build():
    """Called at app startup. Fails fast if dist/ is missing."""
    if not DIST_INDEX.exists():
        raise RuntimeError(
            f"Frontend build not found at {DIST_DIR}. "
            "Run 'cd src/frontend/web && npm run build' first."
        )

# --- ADD: Lifespan handler (replaces any existing on_event("startup")) ---
@asynccontextmanager
async def lifespan(app_instance):
    validate_frontend_build()
    yield

# --- MODIFY: FastAPI instantiation to include lifespan ---
# Change: app = FastAPI(...)
# To:     app = FastAPI(..., lifespan=lifespan)

# --- ADD: Immutable asset cache middleware ---
class ImmutableAssetsCacheMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        response: Response = await call_next(request)
        if request.url.path.startswith("/assets/"):
            response.headers["Cache-Control"] = "public, max-age=31536000, immutable"
        return response

app.add_middleware(ImmutableAssetsCacheMiddleware)

# --- EXISTING: API routes remain unchanged ---
# app.include_router(api_router, prefix="/api")
# These are registered BEFORE the static mount, so they take priority.

# --- ADD: Health check (registered before any catch-all) ---
@app.get("/healthz")
async def healthz():
    return JSONResponse({"status": "ok"})

# --- ADD: Static assets mount ---
if DIST_ASSETS.exists():
    app.mount(
        "/assets",
        StaticFiles(directory=str(DIST_ASSETS)),
        name="static-assets"
    )

# --- ADD: SPA fallback via 404 exception handler ---
@app.exception_handler(StarletteHTTPException)
async def spa_or_api_exception_handler(request: Request, exc: StarletteHTTPException):
    if exc.status_code == 404:
        accept = request.headers.get("accept", "")
        if "text/html" in accept:
            return FileResponse(
                str(DIST_INDEX),
                headers={"Cache-Control": "no-cache"},
                media_type="text/html",
            )
    return JSONResponse(
        {"detail": exc.detail or "Not found"},
        status_code=exc.status_code,
    )
```

**Key design decisions**:
- `/healthz` is a dedicated endpoint returning JSON вЂ” never masked by SPA fallback
- 404 handler uses `Accept` header content negotiation: browsers get `index.html`, API consumers get JSON
- `/assets/*` gets immutable cache headers via middleware since `StaticFiles` doesn't set them
- `index.html` always gets `Cache-Control: no-cache` (browser revalidates on every navigation)
- `validate_frontend_build()` is wired into FastAPI's `lifespan` context manager вЂ” crashes on boot if `dist/` is missing
- Any new backend routes registered via `app.include_router()` or `@app.get()` take priority over the 404 handler вЂ” no swallowing risk

### 2.2 Migration Strategy

**Phase 1 вЂ” Parallel deploy**: Serve React app at `/v2/` path prefix. Vite builds with `base: '/v2/'`. Legacy Alpine app continues at `/`. Backend adds:
```python
if DIST_ASSETS.exists():
    app.mount("/v2/assets", StaticFiles(directory=str(DIST_ASSETS)), name="v2-assets")

@app.get("/v2")
@app.get("/v2/{path:path}")
async def spa_v2_fallback(path: str = ""):
    return FileResponse(str(DIST_INDEX), headers={"Cache-Control": "no-cache"})
```

**API contract during Phase 1**: All API endpoints return relative paths only (no absolute URLs). The React app at `/v2/` calls the same `/api/*` endpoints вЂ” no base-path-aware rewriting needed because API routes are path-absolute (`/api/search`), not relative.

**Phase 2 вЂ” Parity confirmed**: Change Vite `base` back to `'/'`. Swap `/` to serve React via the 404 exception handler. Move legacy Alpine files to `src/frontend/web/_legacy/`. Legacy remains accessible at `/_legacy/` for 2 weeks via a simple `StaticFiles` mount.

**Phase 3 вЂ” Cleanup**: Remove `_legacy/` directory, remove `/v2` routes, remove legacy mount.

### 2.3 Rollback Plan

If React deploy fails after Phase 2 cutover:
1. Revert `web_server.py` to serve `_legacy/` at `/` (single config change)
2. `_legacy/` files are still present for 2-week window вЂ” no rebuild required
3. Rollback is a server restart, not a deploy
