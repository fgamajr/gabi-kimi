# Production Frontend Implementation Spec v4

## 0. Repo Context & Constraints

- **Current state**: Frontend served as static HTML/Alpine/Tailwind from `src/frontend/web/` via `src/backend/apps/web_server.py`
- **Backend endpoints preserved**: `/api/search`, `/api/suggest`, `/api/autocomplete`, `/api/top-searches`, `/api/search-examples`, `/api/document/{doc_id}`
- **Stack**: React 18 functional components/hooks, Tailwind CSS core utilities, pure CSS animations, SVG icons inlined as components, CSS custom properties
- **Breakpoints**: 375px (mobile-first) → 768px (tablet) → 1440px (desktop)
- **Aesthetic**: Premium/confident Nubank + Flighty. Dark-mode-first. No Material/Bootstrap/shadcn defaults, no decorative chrome, no spinners, no ease/linear timing.

---

## 1. Routing Strategy

**Decision: History-based routing with wouter.**

Clean URLs (`/search`, `/document/abc123`) for premium UX, shareable links, and professional appearance. This requires the SPA catch-all in Section 2.

- **Library**: `wouter` v3 — 1.5KB gzipped, history-based by default, no hash routing.
- **Route definitions** live in `app/routes.js` as a flat map of path → lazy component.
- **Navigation direction** is tracked by a `useNavigationDirection` hook (see Section 5.6) that maintains a route-index ordering to determine forward/back. Used solely for screen transition animation selection.
- **No nested routing** — wouter doesn't support it natively and we don't need it. ActionSheet is rendered as an overlay within DocumentScreen (not a child route — see Section 5.3).

**scripts/check-bundle-size.js**:

```javascript
import { readdirSync, statSync } from 'node:fs';
import { join } from 'node:path';
import { execSync } from 'node:child_process';

const DIST = join(import.meta.dirname, '..', 'dist', 'assets');
const BUDGETS = {
  'vendor': { max: 45_000, label: 'Vendor chunk' },
  'index':  { max: 40_000, label: 'Initial app chunk' },
};
const TOTAL_MAX = 160_000;

let total = 0;
let failed = false;

for (const file of readdirSync(DIST)) {
  if (!file.endsWith('.js')) continue;
  const path = join(DIST, file);
  const raw = statSync(path).size;
  const gzipped = parseInt(
    execSync(`gzip -c "${path}" | wc -c`).toString().trim()
  );
  total += gzipped;

  for (const [key, budget] of Object.entries(BUDGETS)) {
    if (file.includes(key) && gzipped > budget.max) {
      console.error(`FAIL: ${budget.label} (${file}) is ${gzipped}B gzipped, budget ${budget.max}B`);
      failed = true;
    }
  }
  console.log(`  ${file}: ${raw}B raw, ${gzipped}B gzipped`);
}

if (total > TOTAL_MAX) {
  console.error(`FAIL: Total JS ${total}B gzipped exceeds budget ${TOTAL_MAX}B`);
  failed = true;
}

process.exit(failed ? 1 : 0);
```

## 2. web_server.py Changes

Minimal, surgical changes to existing server.

### 2.1 Production Serving

```python
import os
from pathlib import Path
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse, JSONResponse
from starlette.exceptions import HTTPException as StarletteHTTPException
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

DIST_DIR = Path("src/frontend/web/dist")
DIST_INDEX = DIST_DIR / "index.html"
DIST_ASSETS = DIST_DIR / "assets"

# --- Startup guard ---
def validate_frontend_build():
    """Called at app startup. Fails fast if dist/ is missing."""
    if not DIST_INDEX.exists():
        raise RuntimeError(
            f"Frontend build not found at {DIST_DIR}. "
            "Run 'cd src/frontend/web && npm run build' first."
        )

# --- Immutable asset cache middleware ---
class ImmutableAssetsCacheMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        response: Response = await call_next(request)
        if request.url.path.startswith("/assets/"):
            response.headers["Cache-Control"] = "public, max-age=31536000, immutable"
        return response

# --- Health check (registered before any catch-all) ---
@app.get("/healthz")
async def healthz():
    return JSONResponse({"status": "ok"})

# --- API routes registered FIRST (existing — no changes) ---
# app.include_router(api_router, prefix="/api")

# --- Add cache middleware ---
app.add_middleware(ImmutableAssetsCacheMiddleware)

# --- Static assets mount ---
app.mount(
    "/assets",
    StaticFiles(directory=str(DIST_ASSETS)),
    name="static-assets"
)

# --- SPA fallback via 404 exception handler ---
# This is safer than a catch-all route: it only fires when no other
# route matched, and only serves index.html to browser navigation
# requests (Accept: text/html). API consumers get proper JSON 404s.
@app.exception_handler(StarletteHTTPException)
async def spa_or_api_exception_handler(request: Request, exc: StarletteHTTPException):
    if exc.status_code == 404:
        accept = request.headers.get("accept", "")
        if "text/html" in accept:
            return FileResponse(
                str(DIST_INDEX),
                headers={"Cache-Control": "no-cache"},
                media_type="text/html",
            )
    return JSONResponse(
        {"detail": exc.detail or "Not found"},
        status_code=exc.status_code,
    )
```

**Key design decisions**:
- `/healthz` is a dedicated endpoint returning JSON — never masked by SPA fallback
- 404 handler uses `Accept` header content negotiation: browsers get `index.html`, API consumers get JSON
- `/assets/*` gets immutable cache headers via middleware since `StaticFiles` doesn't set them
- `index.html` always gets `Cache-Control: no-cache` (browser revalidates on every navigation)
- Any new backend routes (webhooks, metrics, etc.) registered via `app.include_router()` or `@app.get()` take priority over the 404 handler — no swallowing risk

### 2.2 Migration Strategy

**Phase 1 — Parallel deploy**: Serve React app at `/v2/` path prefix. Vite builds with `base: '/v2/'`. Legacy Alpine app continues at `/`. Backend adds:
```python
app.mount("/v2/assets", StaticFiles(directory=str(DIST_ASSETS)), name="v2-assets")

@app.get("/v2")
@app.get("/v2/{path:path}")
async def spa_v2_fallback(path: str = ""):
    return FileResponse(str(DIST_INDEX), headers={"Cache-Control": "no-cache"})
```

**Phase 2 — Parity confirmed**: Change Vite `base` back to `'/'`. Swap `/` to serve React via the 404 exception handler. Move legacy Alpine files to `src/frontend/web/_legacy/`. Legacy remains accessible at `/_legacy/` for 2 weeks via a simple `StaticFiles` mount.

**Phase 3 — Cleanup**: Remove `_legacy/` directory, remove `/v2` routes, remove legacy mount.

### 2.3 Rollback Plan

If React deploy fails after Phase 2 cutover:
1. Revert `web_server.py` to serve `_legacy/` at `/` (single config change)
2. `_legacy/` files are still present for 2-week window — no rebuild required
3. Rollback is a server restart, not a deploy

### 3.7 Focus Ring System

Double-ring technique for guaranteed contrast against any surface:

```css
/* global.css */
*:focus-visible {
  outline: none;
  box-shadow:
    0 0 0 2px var(--color-bg-primary),
    0 0 0 4px var(--color-focus-ring-solid);
  transition: box-shadow var(--duration-fast) var(--ease-out);

@media (prefers-reduced-motion: reduce) {
  *:focus-visible {
    transition: none;
  }
}
```

The inner ring in `--color-bg-primary` creates guaranteed ≥3:1 contrast against the outer `--color-focus-ring-solid` ring on any surface.

### 3.8 Theme FOUC Prevention

`index.html` complete `<head>`:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta name="theme-color" content="#0A0A0B">
  <script>
    (function(){
      var t;
      try { t = localStorage.getItem('theme'); } catch(e) {}
      if (!t) try { t = sessionStorage.getItem('theme'); } catch(e) {}
      if (!t) t = window.matchMedia('(prefers-color-scheme:light)').matches ? 'light' : 'dark';
      document.documentElement.setAttribute('data-theme', t);
      var m = document.querySelector('meta[name="theme-color"]');
      if (m) m.content = t === 'dark' ? '#0A0A0B' : '#FFFFFF';
    })();
  </script>
  <title>App</title>
</head>
<body>
  <div id="root"></div>
  <script type="module" src="/src/main.jsx"></script>
</body>
</html>
```

Script is <350 bytes minified. Runs synchronously before React mounts. `meta[name="theme-color"]` is placed **above** the script. localStorage access is wrapped in try/catch for private browsing; falls back to sessionStorage, then system preference.

## 4. Motion System (`src/styles/animations.css`)

**Non-negotiable rules**:
- Only `transform` and `opacity` animated (compositor-only, no layout thrash)
- No `ease` or `linear` timing — all custom cubic-bezier curves
- All animations gated by `prefers-reduced-motion`
- `will-change` applied only during animation class, removed by JS via `animationend`

### 4.1 Motion Tokens

```css
