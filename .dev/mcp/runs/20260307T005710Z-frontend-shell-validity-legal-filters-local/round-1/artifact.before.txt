Frontend iteration summary:

1. Shared app shell + route-level layout
- App routes now run inside a shared AppShell.
- Desktop: persistent left rail with Home and Search.
- Mobile: persistent bottom nav with Home and Search.
- Document routes keep the shell on desktop and hide the mobile nav.

2. Search experience upgrade
- Search page now includes a structured legal search panel.
- Desktop exposes direct fields for section, art type, issuing organ, and date range.
- Mobile keeps a bottom sheet for the same filter set.
- Search results now surface backend query interpretation via interpreted_query, inferred_filters, and applied_filters.
- Active filter chips include issuing organ.

3. Reader validity/version awareness
- Document API types now include normative_refs and procedure_refs.
- Document page now shows a "Situação normativa" card using cautious heuristics:
  - revocation/replacement signals
  - alteration/complementation signals
  - connected references/procedures
  - no structured references detected
- Document page now shows a compact normative timeline derived from normative references.

4. Existing reader capabilities retained from prior patch
- section parsing + TOC
- reading progress bar
- saved reading position and resume toast
- deep-link restore
- share current position
- copy formal reference
- PDF export fallback
- mobile action bar
- image fallback and secure document renderer remain in place

5. Existing home/dashboard capabilities retained from prior patch
- denser dashboard
- recent history from localStorage
- featured document card
- popular search chips
- operational status card

- Separate real capability gains from cosmetic gains.
