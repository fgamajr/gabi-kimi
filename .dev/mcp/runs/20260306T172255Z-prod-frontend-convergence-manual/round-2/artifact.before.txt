1. BUILD SYSTEM & MODULE STRATEGY
═══════════════════════════════════════════════════════════════

Build tool: esbuild (single dependency, zero-config).
Command: esbuild src/frontend/web/src/main.jsx --bundle --splitting --format=esm --outdir=src/frontend/web/dist --minify --target=es2020
Source code lives in src/frontend/web/src/ (JSX, full React idioms).
Built output lands in src/frontend/web/dist/ (static ES modules, minified, tree-shaken, code-split).
index.html references dist/main.js via <script type="module">.
No dev server. No HMR. No webpack. One deterministic build command.
React 18 and ReactDOM self-hosted in src/frontend/web/vendor/ (no CDN dependency).
Import map in index.html resolves "react" and "react-dom" to local vendor paths.
web_server.py serves src/frontend/web/ unchanged — only static file additions.

3. NAVIGATION & INFORMATION ARCHITECTURE
═══════════════════════════════════════════════════════════════

Bottom tab bar: 3 tabs (Home, Search, Library).
  - Tabs are parallel peer destinations. No drill-down views in tabs.
  - Active tab indicated by accent fill + label; inactive tabs show outline icon only.
  - Tab bar hidden when bottom sheet is open (avoids thumb zone congestion).

Drill-down views (Detail/Timeline): accessed from tapping items on Home or Search results.
  - Back affordance: chevron-left button in top-left header area.
  - Push onto a navigation stack managed by History API.

Routing strategy: hash-based (#/) for zero-config with web_server.py static serving.
  Routes:
    #/              → Home screen
    #/search        → Search screen
    #/library       → Library screen
    #/doc/:id       → Detail/Timeline screen (drill-down, not a tab)

  state/navigation.jsx exports:
    RouterContext: { route: string, params: Record<string,string>, push(path), back() }
    useRoute(): reads context
    <Router>: listens to hashchange, updates context, announces to route-announcer

  Focus management: on route change, move focus to <main> heading (h1) and announce via route-announcer aria-live region.

Bottom sheet: used for primary action flows (new search presets, document actions).
  - Invoked from contextual buttons (not a tab destination).
  - Opens over content, tab bar hides behind overlay.
  - Dismiss: drag handle down, Escape key, overlay tap.
  - Focus trapped inside while open.

Quick actions on list items: explicit "⋯" (more) icon button (44×44 tap target). No long-press. No swipe gestures in v1.

5. DATA TYPES & API CONTRACT
═══════════════════════════════════════════════════════════════

// api.js — all functions accept AbortSignal, return typed promises, throw on non-2xx

type Metric = { id: string, label: string, value: string, trend: "up"|"down"|"flat", delta?: string }
type Status = { state: "operational"|"degraded"|"down", message: string, lastUpdated: string /*ISO*/ }
type Preset = { id: string, label: string, query: string, icon?: string }
type Result = { id: string, title: string, snippet: string, timestamp: string, status: string }
type HistoryEntry = { id: string, query: string, resultCount: number, timestamp: string }
type TimelineEntry = { id: string, timestamp: string, label: string, state: "success"|"warning"|"error"|"info"|"pending", detail?: string }
type Action = { id: string, label: string, icon: string, destructive?: boolean }

fetchMetrics(signal?): Promise<Metric[]>          // GET /api/metrics
fetchStatus(signal?): Promise<Status>              // GET /api/status
fetchPresets(signal?): Promise<Preset[]>           // GET /api/presets
searchDocuments(query, signal?): Promise<Result[]> // GET /api/search?q=
fetchHistory(signal?): Promise<HistoryEntry[]>     // GET /api/history
fetchTimeline(docId, signal?): Promise<TimelineEntry[]>  // GET /api/docs/:id/timeline
fetchActions(docId, signal?): Promise<Action[]>    // GET /api/docs/:id/actions
executeAction(docId, actionId, signal?): Promise<void>   // POST /api/docs/:id/actions/:actionId

Error handling:
- All fetchers throw ApiError({ status, message, retryable }) on failure.
- 8-second timeout via AbortController.timeout(8000) — on timeout, throw with retryable: true.
- Callers catch and render <ErrorState message onRetry /> or <ErrorState message /> (non-retryable).

Autocomplete debounce: 200ms in SearchComposer. Previous in-flight request cancelled via AbortController before new request fires.

6. STATE MANAGEMENT
═══════════════════════════════════════════════════════════════

Global state via React Context + useReducer (no external dependencies):

ThemeContext:
  State: { mode: "dark"|"light" }
  Actions: TOGGLE_THEME
  Persistence: localStorage key "app-theme", fallback to prefers-color-scheme

RouterContext:
  State: { path: string, params: Record<string,string>, stack: string[] }
  Actions: PUSH, BACK, REPLACE
  Sync: hashchange listener in <Router> dispatches updates

DataContext (optional — only if cross-screen data sharing needed):
  State: { metrics: Metric[]|null, status: Status|null, cache: Map<string, {data, timestamp}> }
  Stale-while-revalidate: serve cached data immediately, refetch if cache age > 30s.
  No external library (SWR/React Query). Minimal custom hook: useFetch(key, fetcher, { staleTime }).

Component-local state via useState/useReducer for:
  - Search input value
  - Bottom sheet open/closed
  - Skeleton/loaded/error loading phases
  - Form field values in action sheets

Prop drilling banned past 2 levels. Use context or component composition (render props / children) instead.

8. RESPONSIVE STRATEGY
═══════════════════════════════════════════════════════════════

Approach: Tailwind responsive prefixes (sm:, md:, lg:) mapped to breakpoints.
  - Base (375px+): mobile-first, single column, bottom tab bar.
  - md (768px+): 2-column on Home (KPI rail + status side-by-side). Tab bar remains bottom.
  - lg (1440px+): sidebar nav replaces bottom tab bar. 3-column layout. Max-width 1280px centered.

Fluid typography via clamp():
  --font-display: clamp(1.5rem, 1.2rem + 1vw, 1.875rem);  /* Hero/page titles */
  --font-body: clamp(0.875rem, 0.825rem + 0.25vw, 1rem);   /* Body text */

No container queries in v1 (insufficient Safari support for production). Media queries only.

9. MOTION SYSTEM (animations.css)
═══════════════════════════════════════════════════════════════

All animations use transform + opacity only (GPU-composited, no layout triggers).
All durations reference CSS custom properties.
Aesthetic: Nubank/Flighty — quick, confident, minimal overshoot. No playful/organic motion.

11. SCREEN SPECIFICATIONS
═══════════════════════════════════════════════════════════════

─── HOME SCREEN (#/) ───

Loading phases:
  Phase 1 (immediate, 0ms): Shell renders with header ("Dashboard"), KPI rail skeleton (3 cards: 80×48px rounded-md), status card skeleton (full-width 96px rounded-lg).
  Phase 2 (parallel fetch): fetchMetrics() + fetchStatus() fire simultaneously.
  Phase 3 (on resolve): skeleton → content with .motion-glide + stagger. KPI rail max 3 metrics.
  Phase 4 (deferred, after Phase 3 paint): fetchPresets(), fetchHistory() for lower sections.
  Error: if Phase 2 fails after 8s timeout → replace skeleton with <ErrorState message="Unable to load dashboard" onRetry />.

Skeleton fidelity: skeleton components match exact dimensions of resolved content (KPI card skeleton = same height/width/border-radius as KPIMetric). No generic gray bars.

Content reveal: .motion-glide with .motion-stagger-{1,2,3} on sequential cards.

StatusCard: live-updates via polling (30s interval) with aria-live="polite" region. State renders colored dot (--status-*) + text.

─── SEARCH SCREEN (#/search) ───

SearchComposer: sticky top, input with 200ms debounce, AbortController cancels previous in-flight. Preset chips below input.
Results: list of Result cards, each with title, snippet, timestamp, status badge. Tap navigates to #/doc/:id.
Loading: skeleton list (5 items, each 72px height matching Result card).
Empty: <EmptyState message="No results found" />.
Error: <ErrorState onRetry />.

─── LIBRARY SCREEN (#/library) ───

Chronological list of past searches/interactions.
Loading: skeleton list.
Empty: <EmptyState message="No history yet" />.

─── DETAIL SCREEN (#/doc/:id) ───

Header: sticky, shows document title + status badge. Back chevron top-left.
Timeline: vertical line with TimelineItem nodes. Each node: colored dot (state-based), timestamp, label, optional detail expansion.
Quick actions: "⋯" button in header triggers <BottomSheet> with action list.
Loading: skeleton with sticky header placeholder + 5 timeline item skeletons.
Error: <ErrorState onRetry />.

12. ACCESSIBILITY
═══════════════════════════════════════════════════════════════

WCAG 2.1 AA compliance required.

Hit targets: minimum 44×44px on all interactive elements.
Contrast: all text/bg combinations verified against AA ratio (4.5:1 normal, 3:1 large).
  - Dark: --text-primary (#fafafa) on --surface-0 (#09090b) = 19.4:1 ✓
  - Dark: --text-secondary (#a1a1aa) on --surface-0 (#09090b) = 7.2:1 ✓
  - Light: --text-primary (#09090b) on --surface-0 (#ffffff) = 19.4:1 ✓
  - Light: --accent-default (#7c3aed) on --surface-0 (#ffffff) = 5.4:1 ✓

Semantic HTML: <header>, <main role="main">, <nav> for tab bar, <section> for content regions, <h1>-<h3> hierarchy per screen.
Skip link: "Skip to content" link as first focusable element, visually hidden until focused, targets <main>.
Focus-visible: 2px solid var(--accent-default) outline with 2px offset on all focusable elements. No outline removal.
Focus trap: bottom sheet traps focus. Escape dismisses. Focus returns to trigger element on close.
Route announcer: <div aria-live="assertive" class="sr-only"> announces "Navigated to {page name}" on route change.
Status updates: aria-live="polite" on StatusCard, search result count announcements.
Form errors: aria-live="assertive" + role="alert" on validation error containers in bottom sheet forms.
Reduced motion: all animation disabled via prefers-reduced-motion (see motion system).
